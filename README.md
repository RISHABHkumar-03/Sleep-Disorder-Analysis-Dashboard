# Sleep-Disorder-Analysis-Dashboard
Recommended Structure and Order

1. **Project Title / Headline**
A concise, descriptive name for the dashboard.
Examples: Sleep Disorder Dashboard

2. **short Description / Purpose**
1-2 sentence explaining what the dashboard does and why it exists
Examples:
An interactive Power BI dashboard that analyzes sleep duration, insomnia and apnea cases, and relationships between sleep quality and stress, enabling data-driven insights into sleep health.

3. **Tech Stack**
List the key technologies use to build the dashboard
Examples:
Power BI (data visualization & dashboard development)
DAX (calculated measures and metrics)
Excel / CSV (data source and preprocessing)
Power Query (data transformation)
Python (optional, if used for data cleaning)

4. **Data Source**
More info on where the data comes from how its strucured
Exaples:
This dashboard uses structured sleep dataset(s) containing individual records with fields such as sleep score, sleep duration, disorder types (insomnia, sleep apnea), stress level, age group, BMI category, and occupation. Data was cleaned and prepared in Excel/Power Query before import to Power BI to enable accurate trend analysis and visual reporting.

5. **Features / Hightlights**
The best Dashboard Explatation Format
Examples:
**Interactive Key Metrics**: Displays important sleep health indicators, such as average sleep duration, sleep scores, and case counts for insomnia and sleep apnea.
**Comparative Analysis**: Line charts that compare sleep quality versus stress level across different age groups.
**Categorical Insights**: Bar charts showing distribution of BMI categories and occupation groups for better understanding of underlying patterns.
**Responsive Visualizations**: Easy-to-interpret visuals with consistent color themes and layout for quick insights.
